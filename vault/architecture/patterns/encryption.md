# Encrypted Attributes

Sensitive data that must be encrypted at rest. Uses Laravel's `encrypted:` cast — data is AES-256-CBC encrypted before writing to the database and decrypted transparently on read.

---

## When to Encrypt

Encrypt at-rest when: data leakage from a database dump would cause direct harm or legal liability.

| Field | Domain | Why Encrypt |
|---|---|---|
| `national_id` | HR (Employee) | Identity theft risk; GDPR special category |
| `bank_account_iban` | HR (Payroll), Finance | Financial account data |
| `bank_account_bic` | HR (Payroll), Finance | Financial routing data |
| `date_of_birth` | HR (Employee) | GDPR sensitive personal data |
| `salary` / `gross_pay` | HR (Payroll, Compensation) | Confidentiality sensitive |
| `webhook_secret` | Core (Webhooks) | Credential — leakage enables webhook forgery |
| `api_token_plain` | Core (API Clients) | Token shown once, then stored encrypted |
| `stripe_customer_id` | Core (Billing) | Links to payment data — low risk but best practice |
| `personal_email` | HR (Employee) | Personal contact data, different from work email |

**Do NOT encrypt:**
- `email` (work) — needed for email sending, search indexing, uniqueness constraints
- `first_name`, `last_name` — needed for Meilisearch indexing
- Status fields, dates (other than DOB), foreign keys

---

## Implementation

Use Laravel's built-in `encrypted` cast:

```php
class Employee extends Model
{
    protected $casts = [
        'national_id' => 'encrypted',
        'date_of_birth' => 'encrypted:date', // encrypted + cast as Carbon date
        'personal_email' => 'encrypted',
    ];
}
```

The `APP_KEY` in `.env` is the encryption key. **Never change `APP_KEY` in production** without re-encrypting all encrypted columns first. Key rotation is a separate process.

---

## Searchability Limitation

Encrypted fields cannot be indexed by Meilisearch or queried with `WHERE field = ?`. This is intentional — searching across national IDs is a data privacy violation.

For fields that must be both private and searchable (e.g. look up employee by national ID for payroll reconciliation):

```php
// Store a one-way hash alongside the encrypted value for exact-match lookup
protected $casts = ['national_id' => 'encrypted'];

protected static function boot(): void
{
    parent::boot();

    static::saving(function ($model) {
        if ($model->isDirty('national_id')) {
            $model->national_id_hash = hash('sha256', $model->national_id);
        }
    });
}

// Query by hash
Employee::where('national_id_hash', hash('sha256', $searchTerm))->first();
```

## Salary as Encrypted Integer

Salary stored as encrypted integer cents:

```php
protected $casts = [
    'salary_cents' => 'encrypted:integer', // encrypted, cast as int on read
];
```

Note: `encrypted:integer` is not a native Laravel cast as of Laravel 13. Store salary as an encrypted string representing the integer, then cast manually:

```php
protected $casts = ['salary_raw' => 'encrypted'];

public function getSalaryAttribute(): int
{
    return (int) $this->salary_raw;
}

public function setSalaryAttribute(int $value): void
{
    $this->salary_raw = (string) $value;
}
```

Or use `spatie/laravel-data` with a custom cast in the DTO.

## Key Management

- `APP_KEY` is the encryption key — 32-byte base64 string generated by `php artisan key:generate`
- Stored in `.env` — never committed to git
- In production: stored in the deployment secrets manager (GitHub Actions secrets, DigitalOcean App Platform env vars)
- Key rotation: use `php artisan key:generate --show` to get a new key, then run a one-time command that re-encrypts all encrypted columns with the new key before swapping `APP_KEY`

## Database Migration Example

```php
Schema::create('hr_employees', function (Blueprint $table) {
    $table->ulid('id')->primary();
    $table->ulid('company_id')->references('id')->on('companies');

    // Plaintext — searchable
    $table->string('first_name');
    $table->string('last_name');
    $table->string('email');      // work email

    // Encrypted — stored as text (encrypted value is a long string)
    $table->text('national_id')->nullable();
    $table->text('national_id_hash')->nullable()->index(); // for lookup
    $table->text('date_of_birth')->nullable();
    $table->text('personal_email')->nullable();

    $table->timestamps();
    $table->softDeletes();
    $table->index('company_id');
});
```

Encrypted columns use `text` not `string` — the encrypted ciphertext is longer than 255 characters.

## GDPR Erasure

When a GDPR erasure request is processed, encrypted fields must be overwritten with a null or a replacement value (not just the record soft-deleted):

```php
class EraseEmployeePersonalData
{
    use AsAction;

    public function handle(Employee $employee): void
    {
        $employee->update([
            'national_id' => null,
            'national_id_hash' => null,
            'date_of_birth' => null,
            'personal_email' => null,
            'first_name' => '[Erased]',
            'last_name' => '[Erased]',
            // work email kept for audit trail — anonymised instead
            'email' => "erased_{$employee->id}@flowflex-erased.invalid",
        ]);
    }
}
```
