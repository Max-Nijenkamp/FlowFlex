# Quote PDF Generation

Render a branded PDF of a quote (company logo + colour) via `spatie/laravel-pdf`, queued.

- Generated on send; stored via Media Library ([[../../../core/file-storage/_module]]).
- Line items: description, qty, unit price, discount %, line total (brick/money).
- Runs on the queue ([[../../../../infrastructure/queue-horizon]]).

## UI
- **Kind**: background — PDF render job (`GenerateQuotePdfJob`); no standalone page.
- **Page**: none. The "Download PDF" / "Send" actions live on the quote resource (`QuoteResource` view/table row action).
- **Layout**: no dedicated UI. Job renders the branded template; result is a Media Library attachment surfaced as a download link on the quote infolist.
- **Key interactions**: triggered on `sent` transition (and on-demand "regenerate PDF" action); download from the quote view.
- **States**: empty (no PDF yet — generate action shown) · loading (job queued/running — spinner/"generating…") · error (job failed — retry action) · selected (n/a)
- **Gating**: `crm.quotes.view` to download; PDF generated by the send action (`crm.quotes.send`).

## Data
- Owns / writes: nothing — reads a `crm_quotes` + `crm_quote_lines` row and writes only a Media Library attachment against that quote.
- Reads: `crm_quotes`, `crm_quote_lines`; company branding (logo/colour) from core settings.
- Cross-domain writes: via events only ([[../../../../security/data-ownership]]).

## Relations
- Consumes: `QuoteSent` (within-domain, quotes) → enqueues `GenerateQuotePdfJob`.
- Feeds: nothing cross-domain — output is a stored file on the quote.
- Shared entity: company branding settings (owned by core), read-only.
