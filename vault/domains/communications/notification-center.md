# Notification Center

> Unified inbox aggregating all cross-domain FlowFlex alerts, mentions, and system notifications in one place with preference management.

**Panel:** `comms`
**Module key:** `comms.notification-center`

## What It Does

The Notification Center is the single inbox for every alert generated by any FlowFlex module across the platform. Instead of checking each module separately for outstanding items — a ticket assigned in IT, an invoice approved in Finance, a task overdue in Projects, a contract expiring in Legal — all notifications land in one feed. Users can filter by domain, mark as read, dismiss, or navigate directly to the relevant record from the notification. Notification preferences let each user control which events they receive and through which channels (in-app, email, Slack).

## Features

### Core
- Unified feed: all FlowFlex cross-domain notifications in a single chronological list with the originating domain, event type, and deep link to the source record
- Notification types: assigned (ticket, task, lead assigned to you), mentioned (@mention in a message or comment), action required (approval needed, signature requested), status change (order shipped, contract expiring), alert (anomaly detected, SLA breached)
- Read/unread state: unread notifications highlighted; mark individual or all as read
- Badge count: unread count shown on the notification icon in the main navigation
- Deep link: clicking a notification opens the exact record in the relevant module — no hunting for context
- Dismiss: remove notifications from the feed without marking as read (for low-priority system alerts)

### Advanced
- Filter by domain: filter the notification feed to show only HR, IT, Finance, Sales, etc. notifications
- Filter by type: show only assigned, action required, or mentioned notifications
- Notification grouping: group related notifications (e.g., "3 tasks overdue in Projects") into a single expandable item
- Quiet hours: configure hours when in-app notifications are suppressed; email and Slack digests paused during quiet hours
- Weekly digest: opt-in digest email each Monday morning summarising all outstanding notifications from the prior week
- Notification preferences: per-event-type toggle for each channel (in-app, email, Slack); set granular preferences per domain

### AI-Powered
- Priority sorting: AI scores notifications by estimated urgency (P1 IT incident vs project comment) and surfaces high-priority items at the top
- Noise reduction: learn from dismissal patterns and automatically reduce notification frequency for event types the user consistently dismisses without acting
